# 퍼사드 패턴(Facade Pattern)

## 1. 패턴 이름과 종류
- 이름: 퍼사드(Facade) 패턴
- 종류: 구조(Structural) 패턴

---

## 2. 용도(Intent)
복잡한 서브 시스템을 하나의 간단한 인터페이스 뒤에 숨겨  
사용을 쉽게 만들고 느슨한 결합을 제공하는 패턴입니다.

---

## 3. 동기(Motivation)
홈시어터(Home Theater) 시스템은 DVD 플레이어, 스크린, 앰프 등  
많은 객체를 조합해 동작해야 합니다.  
퍼사드는 “영화 보기 시작하기” 같은 단일 메서드로  
복잡한 절차를 단순화해 줍니다.

---

## 4. 적용 대상(Applicability)
- 복잡한 서브 시스템을 단순화하려고 할 때  
- 서브 시스템 간 의존성을 줄이고 싶을 때  
- 라이브러리 사용법을 쉽게 제공하고 싶을 때

---

## 5. 구조(Structure)

```

+-------------------+
|      Client       |
+-------------------+
         |
         v
+-------------------+
|      Facade       |
+-------------------+
| + simpleOperation()   |
+-------------------+
   |        |        |
   v        v        v
+------+ +------+ +------+
|Sub 1| |Sub 2| |Sub 3|
+------+ +------+ +------+

(Facade 는 여러 Subsystem 클래스들을 안쪽에서 조합해
단순한 메서드 여러 개로 감싸서 제공)

```

---

## 6. 구성 요소
- **Facade**: 단순화된 인터페이스 제공  
- **Subsystem 클래스들**: 복잡한 실제 기능 제공

---

## 7. 협동(Collaborations)
Facade는 여러 서브 시스템의 기능을 조합하여  
클라이언트가 더 간단하게 호출할 수 있게 해줍니다.

---

## 8. 결과(Consequences)
### 장점
- 인터페이스 단순화  
- 서브시스템과 클라이언트의 결합도 감소  
- 유지보수성 증가

### 단점
- Facade 자체가 너무 많은 기능을 갖게 되면 또 다른 복잡도 발생

---

## 9. 구현(Implementation)
- Facade는 최소한의 기능만 제공  
- 서브 시스템 클래스는 그대로 노출되어도 무방

---

## 10. 샘플 코드(Home Theater)

```java
public class HomeTheaterFacade {
    private DVDPlayer dvd;
    private Amplifier amp;

    public HomeTheaterFacade(DVDPlayer dvd, Amplifier amp) {
        this.dvd = dvd;
        this.amp = amp;
    }

    public void watchMovie() {
        amp.on();
        dvd.on();
        dvd.play();
    }

    public void endMovie() {
        dvd.off();
        amp.off();
    }
}
```

## 11. 사용 예
-Spring의 JdbcTemplate
- Servlet의 RequestDispatcher
- Hibernate SessionFactory

## 12. 연관 패턴
- 어댑터 패턴: 인터페이스 변환 목적
- 추상 팩토리: 서브시스템 객체 생성 담당
